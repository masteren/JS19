JavaScript オリジナル作品課題

学習した JavaScript の技術を活かして、オリジナルの Web ページを作ろう

「オリジナル Web ページ・アプリ」を JavaScript で作ろう

テーマは完全自由です。好きなもの・興味のあることを題材に、動くページを作ることを最優先。 バックエンドのオリジナル作成、外部 API を利用してもOK。

好きなアーティスト紹介 おすすめスポット一覧 ゲームキャラ図鑑 料理レシピページ ポートフォリオ など自由に！
フロントエンドは React や Vue などのフレームワークは使わず、HTML・CSS・JavaScript で作成してください。

評定基準と注意点

1

評定基準

提出された作品は、以下の観点で評価されます。

  • 下記の要件を満たしているか（必須項目の達成度）
  • 授業でやったような内容を活用しているか
  • 独学で学んだ技術やアイデアに自主的に挑戦したか

2

フォルダ・ファイルを作成する

プロジェクト用フォルダを作り、ファイルを用意。
REAEDME.md には、作品テーマ・使用技術・工夫した点・AIの活用方法などを簡単にまとめる。

ファイル構成の例

my-project/
├── index.html
├── css/style.css
├── js/app.js
└── README.md
3

AIコーディングの活用と注意点

AIを活用してアイデア出しやコードのサンプルをもらいながら作成を進めてみましょう。
ただし、AIを利用する場合は、以下の点に注意してください。

  • AIはあくまで補助ツールであり、最終的な作品は自分の手で作ることが重要。
  • AIから得たアイデアやコードは、自分の作品に合うようにアレンジ・カスタマイズする。
  • AIを使用した場合は、どのように活用したかを作品の説明に記載。
    例：「アイデア出しにChatGPTを使用しました」
    例：「APIの呼び出しコードはAIに生成してもらい、そこから自分で改良を加えました」

Topics — 今までやった学習内容

メニュー

ハンバーガーメニュー、ドロップダウン、タブ切り替えなど、クリックで表示・非表示を制御する UI

イメージビュー

モーダルで画像を拡大表示、スライドショー、クリックで画像を切り替えるギャラリーなど

アニメーション・パララックス

CSS トランジション・キーフレームアニメーション、スクロール連動など

スクロール量に応じて背景・要素を異なる速度で動かす視差効果

API（Fetch API・JSON）

fetch() で外部 API からデータを取得し、JSON をパースしてページに表示する

Requirements — 要件

必須 ここまでが提出の最低条件です
01

HTML・CSS・JavaScript の 3ファイルで構成されている

index.html をブラウザで開いて動作すること。外部フレームワーク不要。

02

JavaScript でページの動作を制御している

クリック・スクロールなどのイベントに反応して、何らかの変化が起きること

03

Webサーバで動作すること

Webサーバ（http://localhost など）を想定して動作確認すること。
Webアプリの多くは、JavaScript でファイル操作や API 呼び出しを行うため。

加点 挑戦してみよう

API を使ってデータを取得・更新

外部 API（天気、Pokémon API など）を fetch() で呼び出してページに表示している

コーディング

機能ごとに関数を分けたり、複数の .js ファイルに分割するなど、コードが読みやすい構成になっている

デザイン

レイアウト・配色・フォントなどが整理されており、テーマに合ったデザインになっている

その他工夫している点がある

バックエンドとの連携、ローカルストレージ活用、Canvasを利用したドロー、オリジナルのインタラクション、アクセシビリティへの配慮など
